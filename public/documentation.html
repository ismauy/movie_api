myFlix Project


Documentation Page

The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Request Business Logic Url Request body format Response format
GET Get a list of all movies /movies Return list of movies - JSON objects.
GET Get data about a movie by title /movies/:title Specify Title in URL Return data about a single movie - JSON objects,
Example:

{
"Genre":
"Name": "Genre's name",
"Description": "Description of the genre.",
"Director: "
"Name": "Director's Name",
"Bio": "Director's biography",
"Birthyear": "YYYY",
"Deathyear": "YYYY",
"_id": objectId,
"Title": "Movie Name",
"Description": "Movie description",
"imageUrl": Soon
"Featured": True
}

GET Get data about a Genre /genres/:name Specify Genre name in URL Return data about a single genre - JSON objects,
Example:

{
"Name": "Genre's name",
"Description": "Description of the genre.",
}

GET Get data about a director by name /directors/:name Specify Director name in URL Return data about a single movie director - JSON object
Example:

{
"Name": "Director's Name",
"Bio": "Director's biography",
"Birthyear": "YYYY",
"Deathyear": "YYYY"
}

POST Add a new user /users Data structure about the new user to add

{
"Username": "Username",
"Password": "Password",
"Email": "user@user.com",
"Birthday": "yyyy-mm-dd",
}

Return data about the new user added - JSON object

{
"Username": "Username",
"Password": "Password",
"Email": "user@user.com",
"Birthday": "yyyy-mm-dd",
"_id": objectId,
}

PATCH Allow users to add a movie to favorite list /users/:id/favorites/:title Return data of a user and its favorites movies that is sucessfully updated - JSON object
Example:

{
"_id": objectId,
"Username": "Username",
"Password": "Password",
"Email": "user@user.com",
"Birthday": "yyyy-mm-dd",
"FavoritesMovies": [movieId]
}

PUT Update a certain user by username /users/:id Data example about user info to update,

{
"Email": "user-updated@mail.com", }

Return data of a user info that is sucessfully updated - JSON object
Example:

{
"_id": objectId,
"Username": "Username",
"Password": "Password",
"Email": "user-updated@mail.com",
"Birthday": "yyyy-mm-dd",
"FavoritesMovies": [movieId]
}

DELETE Allow users to remove a movie from their favorite list /users/:id/favorites/:title Return TEXT "movie removed".
DELETE Allow users to de-register /users/:id Return TEXT "user 'email' has been de-registered".
